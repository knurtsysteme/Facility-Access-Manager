How to edit agreements

As an Admin you can define agreements on $util.html.a("editsoa").setContent("this page").

Features:

  • Each role can have a different set of agreements
  • Each agreement may have a print flag, so the user must print out that agreement
  • The area provide an overview of the current configuration as well as a history of configurations

Usage

There are five different panels:

  • View existing agreements: Shows all agreements ever prepared.
  • Prepare new agreement: Create a new agreement using an existing agreement as template.
  • Set agreements: Configuration and activation of agreements
  • Current configuration: View the current configuration
  • History configurations: View the history configuration

View existing agreements

This is a pool of all agreements the system knows. An agreement in this view can be activated, deactivated or never has been used.

To activate an agreement follow the instructions of “Set agreements”

Prepare new agreement

Because of transparency as related to the question “Who had when which agreement?”, it is not possible to edit agreements. Use an existing agreement as a Draft instead. Of course you can start writing a new agreement without a draft es well. As an editor the system uses tinymce (see Wiki for details)

You may want to use placeholders like $user in your agreement. Please find the complete documentation here. For all your definitions you can use the template engine Velocity.

After clicking button “Insert this Terms of use Agreements”, it is possible to activate it on tab “Set agreements”.

Without activating your new agreement your input does not have any effect

Set agreements

Refer existing agreements to users with a specific role, configure the order, define the necessity of printing for each agreement and activate the configuration here. Just follow the steps as described.

After clicking “Set this new configuration” all users with the configured role must accept the new agreements immediately before they are able to continue working with the system.

The agreemets used for this role so far gets deactivated (can be find in Tab “History configurations”)

Current configuration

Get an overview of what role must currently accept which agreements.

History configurations

Get an overview of what role must accept which agreements in past.
